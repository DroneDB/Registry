# QGIS smoke-test project

This folder is the place to commit a `.qgz` project that pre-configures the
WMS / WFS / WMTS / OGC API connections produced by `scripts/qgis-test-setup.ps1`.

Because a QGIS project file (`.qgz`) is a binary zip and must be created from
QGIS itself, it is **not** generated by these scripts.

## How to (re)build it

1. Run `..\qgis-test-setup.ps1` (or `.sh`) and copy the printed endpoints.
2. Open QGIS, add WMS/WFS/WMTS/OGC API connections pointing at those URLs
   following the step-by-step guide in
   [`DroneDB-Docs/docs/ogc-services.md`](../../../DroneDB-Docs/docs/ogc-services.md).
3. Save the project as `ogc-test.qgz` next to this README.

The harness assumes the URL pattern below (so you can re-bind the project to
any Registry instance by editing the connection):

```
http://localhost:7000/orgs/qgis-test/ds/ogc-fixture/{service}
```

Where `{service}` is `wms`, `wfs`, `wmts`, `wcs`, `ogcapi/features`, `ogcapi/tiles`
or `mvt/{hash}/{z}/{x}/{y}.pbf`.
